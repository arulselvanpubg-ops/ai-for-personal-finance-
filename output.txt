■ 1. Executive Summary
FinSight AI is a Python-based, desktop/web personal finance assistant that leverages three best-in-class free
AI APIs — NVIDIA NIM, Hugging Face, and Google AI Studio — to deliver intelligent budgeting, expense
analysis, investment insights, and a conversational LLM chatbot. Built for vibe-coding in VS Code, FinSight
targets tech-savvy individuals who want an offline-capable, privacy-first financial co-pilot.
Traditional budgeting apps deliver passive reports. FinSight AI acts as an active financial coach — categorizing
expenses automatically, forecasting cash flow, detecting anomalous spending, and answering plain-language
financial questions in real time. The system is entirely open-source-stack, zero subscription cost (API free tiers),
and designed for local-first data storage so users retain full control.

■ 2. Project Overview & Goals
2.1 Problem Statement
Over 37% of Americans already use AI to help manage finances (Ipsos/BMO, 2025), yet most tools are
cloud-locked, subscription-based, or generic. FinSight AI fills the gap: a powerful, locally-run assistant with real
intelligence, not just dashboards.

2.2 Goals
• Provide an AI-powered personal finance dashboard with smart insights.
• Deliver an LLM chatbot (FinSight Chat) for conversational money management.
• Automate expense categorization using HuggingFace NLP models.
• Generate predictive cash-flow forecasts with Google AI Studio Gemini.
• Keep 100% of financial data local — no cloud sync required.
• Achieve a clean, modern UI with a teal/amber palette (non-generic design).
• Ensure the full stack is Python — easy to vibe-code and extend in VS Code.

■ 4. AI Architecture & API Integration
4.1 NVIDIA NIM Free API (Primary LLM)
Endpoint: https://integrate.api.nvidia.com/v1
Model: meta/llama-3.1-70b-instruct (or nvidia/nemotron variants)
Free Tier: Available at build.nvidia.com — no credit card required for dev
Role: Powers FinSight Chat — the conversational financial assistant chatbot. Handles follow-up questions,
spending advice, and budgeting suggestions in natural language.
Sample usage pattern (Python):
from openai import OpenAI
client = OpenAI(base_url='https://integrate.api.nvidia.com/v1', api_key=NVIDIA_API_KEY)
resp = client.chat.completions.create(
model='meta/llama-3.1-70b-instruct',
messages=[{'role':'user','content': user_prompt}],
stream=True
)

4.2 Hugging Face API

Role: Expense auto-categorization NLP, semantic similarity, embeddings.

Free Tier: Inference API – 30,000 requests/month free.
• Zero-shot classify each transaction description into budget categories (Food, Rent, Transport, etc.).
• Generate sentence embeddings for semantic search over past transactions.
• Optional: Fine-tune a small model locally on the user's own spending history.

4.3 Google AI Studio API (Gemini)
Role: Financial insight generation, monthly summaries, cash-flow forecasting.
Model: gemini-1.5-flash (free tier: 1,500 req/day, 1M tokens/min)
Free Tier: aistudio.google.com — no billing required.
• Generate plain-English monthly financial health summaries.
• Produce 3-month cash-flow forecasts based on spending trends.
• Detect recurring subscriptions and flag unusually high spend categories.
• Answer complex financial queries that require multi-step reasoning.

4.4 AI Routing Logic
The system uses a lightweight router to direct requests to the most suitable API. Conversational dialogue →
NVIDIA NIM. Categorization/embeddings → Hugging Face. Analysis/forecasting → Google Gemini. This
maximises free-tier usage and keeps latency low by matching task type to model strength.

■ 6. LLM Chatbot — FinSight Chat

FinSight Chat is a persistent, context-aware financial chatbot powered by NVIDIA NIM (Llama 3.1 70B). It
maintains a rolling conversation window, has access to a summarized view of the user's financial data, and
responds with actionable, personalised advice.

6.1 System Prompt Design
A carefully engineered system prompt is injected at every session start. It includes: the user's current month
spending summary, top 3 budget overruns, savings goal progress, and a persona definition (friendly financial
advisor, no jargon). This grounds the LLM with real user data without sending raw transaction rows.

6.3 Streaming UI
Responses stream token-by-token into the chat window using NVIDIA NIM's streaming mode (stream=True). A
typing indicator animation plays while the first token is awaited. The chat history is stored in SQLite
(session-based) and the last 10 turns are included in each API call for context continuity.

■ 9. Security & Privacy
• Local-first data: All transactions stored in SQLite on the user's machine. No cloud sync.
• API keys in .env: Never hardcoded. python-dotenv loads from .env file. .env added to .gitignore.
• Encryption at rest: Optional SQLite encryption via SQLCipher (sqlcipher3 library).
• No PII to LLMs: Only anonymized summaries (totals, category names) are sent to AI APIs — never raw
transaction descriptions with merchant names.
• HTTPS only: All API calls use TLS. Certificate verification enforced.
• Input validation: All user inputs sanitized before DB writes (parameterized queries, no raw SQL).
• Session timeout: App auto-locks after 15 minutes of inactivity (configurable).

■ 16. Glossary
NIM: NVIDIA Inference Microservice — hosted LLM endpoints from NVIDIA.
HuggingFace Inference API: Free hosted API to run ML models in the cloud.
Gemini: Google's multimodal LLM family; Flash variant is free-tier.
RAG: Retrieval-Augmented Generation — grounding LLM answers with local data.
Zero-shot classification: Categorizing text without fine-tuning, using NLI models.
SQLite: Serverless, file-based relational database. No installation needed.
Streamlit: Python framework for building interactive data apps rapidly.
pdfplumber: Python library for extracting text/tables from PDF files.
yfinance: Unofficial Yahoo Finance Python API for stock price data.
SIP: Systematic Investment Plan — periodic fixed investment in funds/stocks.
WCAG 2.1 AA: Web Content Accessibility Guidelines — contrast and usability standard.
